# agentSec 项目需求文档

> 状态：已确认  
> 依据：Phase 1 讨论 + PRD/UI 定稿回补（资产管理增量）+ Phase 2 Step A 性能/存储共识  
> 关联 PRD：[`docs/design/prd-sketch.md`](../design/prd-sketch.md)

---

## 一、项目背景

### 背景说明

本机 Agent（Hermes、OpenClaw 等）广泛使用 MCP、Skills、知识库与依赖组件，普通用户难以判断配置是否安全、暴露面是否过大、依赖是否存在已知漏洞。

### 问题描述

用户不知道本机 Agent 暴露了哪些能力（工具、权限、知识库路径等），也不知道这些暴露面风险等级如何，更难以管理 MCP/Skills 的更新与禁用。

### 项目目标

交付一款 **桌面安全工具**：一键扫描本机 Agent 的暴露面/基线风险与组件 CVE，并在同一应用内 **查看与管理** Agent 资产。

### 非目标

- 企业多租户与集中管控平台
- 全自动渗透 / exploit 验证 / Prompt 投毒 payload 跑通
- 攻击路径多步图谱推理（MVP）
- 自动修复配置（须用户明确触发管理操作）
- 扫描历史对比 / Diff
- 定时后台扫描与实时行为告警（vNext）

---

## 二、用户分析

### 目标用户

| 用户 | 特征 | 核心诉求 |
|------|------|----------|
| 普通 Agent 用户 | 使用 Hermes / OpenClaw 等本机 Agent | 不知道本机 Agent 是否安全 |
| 有一定动手能力的个人用户 | 能理解安装、更新、禁用、卸载 | 想知道装了哪些 MCP/Skills，能否管理 |
| 小团队成员 | 使用相似 Agent 配置 | 快速定位高风险配置与过期组件 |

MVP **不以**安全研究员、企业安全平台管理员为主用户。

## 三、核心场景

| 场景 | 说明 | 优先级 |
|------|------|--------|
| S4 本机 Agent 定期体检 | 一键扫描本机 Hermes + OpenClaw，查看风险与 CVE | P0 |
| S3 指定路径扫描 | 通过隐藏入口选择本机全部或自定义路径 | P1 |
| 资产管理 | 按 Agent 查看 MCP/Skills/知识库/依赖，执行更新/禁用/卸载 | P0 |
| 风险复核 | 查看高/中/低分级与影响说明，人工决定是否处理 | P0 |

---

## 四、核心痛点

| 痛点 | 当前方案 | 存在问题 |
|------|----------|----------|
| P1 不知道暴露了什么 | 手动翻配置文件 | 分散、无汇总、无风险语义 |
| P2 不知道危不危险 | 凭经验判断 | 缺少分级与影响说明 |
| P3 不知道如何改 | 搜索文档 | MVP 后置完整修复指引；须先有 severity + 影响说明 |
| P4 团队统一治理 | 无 | MVP 排除 |

MVP 覆盖 **P1 + P2**；P3 完整修复指引后置，但威胁输出须含 severity 与影响说明。

## 五、价值主张

### 核心价值

> Agent 版安全卫士：一键扫描本机 Agent 的暴露面、Prompt Injection 风险与组件漏洞，并管理相关资产。

### 差异化优势

| 维度 | 说明 |
|------|------|
| Agent 专用 | 从 Agent 配置、MCP 权限、Skills、知识库路径做暴露面检测，非传统 SAST |
| 双管线 | 暴露面/基线（规则引擎）与组件 CVE（SCA）分开统计与展示 |
| 一站式本机 | 默认 `scan --all` 发现 Hermes + OpenClaw，架构可扩展更多 Adapter |
| 可管理 | 不仅发现资产，还支持查看、更新、禁用、卸载（用户触发） |

### 替代方案或竞品分析

通用 SCA（Trivy/Grype）不覆盖 Agent 暴露面；IDE 配置审计工具通常单产品、缺统一管理台体验。

## 六、MVP 范围

### 6.1 包含（Must）

#### 安全扫描

| ID | 能力 | 说明 |
|----|------|------|
| M-S1 | 一键扫描 | 默认全量扫描，不提供扫描类型勾选 |
| M-S2 | 自定义扫描路径 | 首页隐藏入口；弹窗选择本机全部 / 自定义路径 |
| M-S3 | Adapter | Hermes + OpenClaw 发现与解析 |
| M-S4 | 暴露面与基线 | 配置、权限、Prompt 注入面等；高/中/低 + 影响说明 |
| M-S5 | Prompt Injection 检测 | 注入面 + 规则检测；**非** exploit 验证 |
| M-S6 | 组件 CVE | 依赖提取 + 版本匹配 + 升级建议 |
| M-S7 | 识别引擎 | **暴露面**：**ATR**（本地 rules + pyATR，扫描时纯离线）；**CVE**：联网 OSV（无网不可用） |

#### 资产管理（Phase 1.5 / 设计定稿回补）

| ID | 能力 | 说明 |
|----|------|------|
| M-AM1 | 独立模块 | 侧栏「资产管理」与安全扫描并列；Step3 资产卡可进入 |
| M-AM2 | Agent 列表 | Step6：Hermes/OpenClaw 卡片 + 进入工作台 |
| M-AM3 | Agent 工作台 | Step7：Tab 概览 / 权限管理 / 资产管理 / **态势图谱** / 威胁管理 / 漏洞管理 |
| M-AM4 | 组件查看 | 用途、来源、权限、状态、版本 |
| M-AM5 | 管理操作 | 更新、禁用、卸载；危险操作二次确认 |
| M-AM6 | 权限聚合 | 概览权限雷达 + 隐蔽入口 → 权限详情弹窗；多来源聚合 + **定位来源**（非固定跳 MCP） |
| M-AM7 | 数据复用 | 扫描 Discovery 写入**最近一次**快照；资产管理读快照，打开 Step7 不触发全量重扫 |
| M-AM8 | 态势图谱 | Agent 工作台独立 Tab「态势图谱」：2D 配置关系图（Agent → 资产类型/组件 → 暴露面/CVE）；与资产管理/威胁/漏洞共用快照；支持筛选、节点详情与跳转 |

**操作能力说明：** MVP 以各 Agent/Adapter **实际支持**为准；不支持的操作灰显并说明原因，不承诺四类操作在所有 Agent 上对称可用。

#### 交付与设置

| ID | 能力 | 说明 |
|----|------|------|
| M-D1 | 桌面客户端 | 暗紫毛玻璃 UI；设置放应用内（语言、主题、扫描、确认策略） |
| M-D2 | 结果展示 | 暴露面与 CVE **双 KPI** 分开；详情左右分栏，不用详情弹窗（权限详情弹窗除外） |

### 6.2 不包含（Won't）

| 能力 | 说明 |
|------|------|
| 攻击路径多步推理 | 多步攻击链自动推理与 exploit 路径演算 |
| 态势关系可视化 | ~~不做~~ → **v0.2+ 已纳入**：单 Agent 配置关系图（非 3D、非多步推理） |
| 自动修复 | 不自动改配置 |
| 扫描历史 / Diff | 仅保留最近一次 |
| 定时后台扫描 | 无计划任务/守护进程 |
| 企业多租户 | 无团队集中管控 |
| 实时行为告警 | vNext |
| Cursor 等 Adapter | MVP 仅 Hermes + OpenClaw |

## 七、成功标准

### 用户指标

1. 打开应用后理解「这是检查 Agent 安全风险的工具」
2. 一键完成本机安全扫描
3. 区分暴露面/基线风险与组件漏洞
4. 查看风险/CVE 的影响与建议
5. 找到本机 MCP/Skills 并理解用途
6. 对组件执行更新、禁用、卸载（在 Adapter 支持范围内）

### 业务指标

MVP 阶段以可交付桌面版与 2 个 Adapter 可用为主，不设商业 KPI。

### 技术指标（Phase 2 Step A 已确认）

| ID | 指标 |
|----|------|
| NF-P1 | 单 Agent 全量扫描 ≤ **1 分钟** |
| NF-P2 | `scan --all`（2 Agent）≤ **2 分钟** |
| NF-P3 | 扫描开始后 ≤ **10 秒** 出现进度与第一批 Finding |
| NF-P4 | 扫描后台执行；UI 可切页、不冻结；支持取消 |
| NF-D1 | **仅保留最近一次**扫描结果；**重启应用仍可查看** |
| NF-D2 | 本地 CVE 精简库目标 ~**50MB** 量级（可联网补全，不阻塞主结果） |
| NF-A1 | 扫描**未完成**（含取消、崩溃）：**不展示本次结果**，丢弃进行中数据；仍保留**上一次完整成功**的快照 |
| NF-A2 | **CVE 联网失败时，CVE 检测不可用**（不展示 CVE 结果；暴露面结果不受影响） |
| NF-A3 | 资产管理写操作失败：**弹框提示**错误原因；禁止静默失败或部分写坏配置 |
| NF-S1 | 数据 **纯本地**：扫描结果、缓存、日志不出本机；MVP **无**遥测、无云同步、**无**报告导出 |
| NF-S2 | **落盘脱敏**：不存完整 secret；凭证类仅存「检测到引用」+ 位置；UI 展示 Key 类证据可打码（如前后 4 位） |
| NF-S3 | 应用日志默认不记录扫描内容明细，仅阶段/错误码 |

### 团队与栈倾向（Phase 2 Step A，非最终 ADR）

| 项 | 结论 |
|----|------|
| 团队规模 | **1 人**（MVP  solo） |
| 核心引擎 | **Python**（Discovery、规则引擎、SCA、Adapter） |
| 桌面 UI | **Electron + React/Vite + TypeScript** + Python 引擎 sidecar |
| 交付顺序 | **macOS 先行**，Windows 后续 |

### 运行环境与分发（Phase 2 Step A）

| 项 | 结论 |
|----|------|
| OS | macOS **13+**（Apple Silicon / Intel） |
| 内存 | 目标机器空闲 **≥ 4GB**；扫描峰值 app 占用目标 **≤ 512MB** |
| 磁盘 | 安装包 + CVE 缓存 + 快照总量 **≤ 300MB** |
| 网络 | 暴露面扫描 **离线**；CVE **必须联网**（失败则不可用） |
| 分发 | MVP **仅 dmg** 本机安装；不走 App Store |

## 八、关键假设与风险

| 假设 | 状态 |
|------|------|
| Hermes/OpenClaw 配置可静态读取 | ⚠ 待 Adapter 调研验证 |
| 更新/禁用/卸载可通过配置或 CLI 安全写回 | ⚠ 待 Adapter 调研；可能部分灰显 |
| 单 Agent MCP/Skill ≤20、依赖 ≤500 为典型规模 | ✅ 架构默认假设 |
| 用户接受最近一次结果、无历史 Diff | ✅ |

## 九、讨论记录

### 已达成共识

- 主用户：普通本机 Agent 用户（兼小团队）
- MVP Adapter：Hermes + OpenClaw
- 识别引擎：暴露面 **ATR**（离线）；CVE **OSV 联网**；OpenClaw 官方 audit 作补充
- 交付形态：桌面客户端（非 CLI+Web 首版）
- 资产管理：独立 Must 模块；Step6→Step7；扫描写快照、管理读快照
- 性能/存储：见第七节技术指标
- 可用性：扫描未完成全部丢弃；CVE 联网失败则 CVE 不可用；写失败弹框提示
- 安全：纯本地、无导出；落盘脱敏
- 团队/栈：1 人；Python 核心；Electron + React/Vite + TS
- 环境/分发：macOS 13+；≤300MB 磁盘；仅 dmg
- 上下文 B1：组件更新经 **npm/pip 等包管理器**（下游依赖）；**不**管理 npm 缓存
- 流程 B3：写成功后 **增量 patch 快照**（B3-b）；10s 内 **仅进度+资产数**（3-a）
- 数据 B4：SnapshotStore 唯一读源；patch 只改资产/权限/统计，Finding 靠重扫
- CVE 演进：MVP **联网 OSV**；vNext 本地 CVE 库/缓存
- 暴露面引擎（2026-06 修订）：**ATR**（pyATR + 内置 rules 子集）；见 [`docs/engine/atr-mvp-rules.md`](../engine/atr-mvp-rules.md)

### 存在分歧

| 项 | 说明 |
|----|------|
| CVE 离线策略 | MVP：**联网 OSV 实时查**；无网 CVE 不可用。vNext：本地 CVE 库/缓存（已纳入架构演进，非 MVP） |

### 待验证事项

- OpenClaw/Hermes 各管理操作（更新/禁用/卸载）的实际 API 与权限边界
- CVE 精简库裁剪范围与更新频率
- npm/pip 更新失败时的错误语义与用户提示
